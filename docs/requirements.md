# 不動産価格予測システム 要件定義書

## 1. 概要

中古マンションの予測価格を算出するWebアプリケーションを開発する。

本プロジェクトは不動産価格予測を題材として以下を実践することを目的とする。

* 機械学習
* システム設計
* データエンジニアリング
* 特徴量設計
* モデル運用

学習済みモデルをブラウザ上で実行し、サーバーレスで動作する構成を採用する。

## データ取得

* 自動取得スクリプトを作成する

取得元は国土交通省 不動産情報ライブラリを利用する。


API利用時はAPIキーを利用し、学習側のcollect処理から取得する。

# 5. 入力仕様

## 地図入力

ユーザーは地図をクリックして位置を指定する。

取得項目

* 都道府県
* 市区町村
* 最寄駅
* 緯度
* 経度

### MVP

逆ジオコーディングAPIで都道府県・市区町村を取得する。

最寄駅は緯度経度と駅マスタCSVの距離計算により決定する。

MVPでは総当たり探索を利用する。

### 将来構想

GeoJSONを利用したブラウザ内判定へ移行する。

目的

* API依存の削減
* 完全サーバーレス化
* GitHub Pages単独運用

# 6. 出力仕様

## 予測結果

* 予測価格
* 平米単価
* 信頼区間

## 価格推移

### 過去価格推移

学習データを集計して表示する。

学習データ全件をブラウザへ配布せず、学習時に生成した集計済みJSONを利用する。

集計単位

* 年単位
* 最寄駅単位

### 将来価格予測

ユーザー指定年に対して予測を行う。

将来予測は参考値であり、精度保証は行わない。

将来予測は価格上昇率モデルではなく、transaction_year を特徴量として学習した結果を利用する。長期予測の精度は保証しない。

学習データの最新年から10年を超える予測年では警告を表示する。

# 13. 評価指標

## 主要指標

* MAE

## 補助指標

* RMSE
* MAPE

## MAE採用理由

利用者が知りたいのは平均して何万円程度の誤差が発生するかである。

MAEは価格予測において解釈しやすく、ユーザー価値との対応関係が明確であるため主要指標として採用する。
